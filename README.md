# 🤖 Dual-Agent tmux Collaboration Project (OMX & Gemini)

이 프로젝트는 **tmux** 환경 내에서 코드를 구현하는 **OMX(Codex)** 에이전트와 코드 품질을 감시하는 **Gemini(code-reviewer)** 에이전트를 유기적으로 협업시켜, 기획부터 개발, 코드 리뷰까지의 전체 개발 라이프사이클을 극대화하는 자동화 개발 템플릿입니다.

---

## ⚡ 1. 아키텍처 및 역할 분담

이 환경은 좌우 분할 창(Pane)을 통해 두 에이전트에게 비대칭적인 권한과 역할을 부여합니다.

*   **좌측 Pane (OMX / Codex)**: **"실제 기능 구현관 (손)"**
    *   **권한**: 읽기/쓰기/파일 수정/터미널 명령 실행 권한 활성화.
    *   **역할**: 명세에 맞춘 실제 소스 코드 작성 및 패키지 설치.
*   **우측 Pane (Gemini / code-reviewer)**: **"수석 코드 검토관 (머리)"**
    *   **권한**: 오직 **읽기 전용(Read-only)**. 파일 수정 및 쓰기 권한 강력 차단 (샌드박스 격리).
    *   **역할**: 코드 스타일 및 성능 리뷰, 보안 결함 체크, 잠재 버그 정적 분석.

## 🚀 2. 평소 개발 루프 (Daily Coding Loop)

### Step 1. 개발 런타임 세션 켜기
터미널에서 아래 명령을 쳐서 tmux 세션을 시작합니다. 화면이 즉시 좌우로 분할되며 두 에이전트가 마주 보고 구동됩니다.
```bash
./scripts/start-ai.sh gemini
```

### Step 2. 창 오가기 (tmux 단축키)
*   **`Ctrl + b` 누른 뒤 `방향키(← / →)`**를 사용해 좌우 창 포커스를 신속히 전환합니다.

### Step 3. 역할에 맞춰 직접 타이핑하여 협업하기
*   **개발 구현 시키기 (좌측 창)**: 접두사 없이 대화하듯 편하게 코딩을 시킵니다.
    ```text
    app.js 서버 파일 생성하고 3000포트로 열리게 짜줘
    ```
*   **코드 검토 시키기 (우측 창)**: 접두사 없이 편하게 리뷰를 시킵니다.
    ```text
    방금 구현된 app.js 파일에 버그가 없는지 검토하고 리뷰해줘
    ```
*   **새로운 기획서 작성시키기 (우측 창)**: 검토관을 임시로 기획자로 변신시킵니다.
    ```text
    $product-manager 실시간 가계부 서비스 스펙 기획서 써줘
    ```
*   **프로젝트 아키텍처 설계시키기 (우측 창)**:
    ```text
    $architect 기획안 바탕으로 폴더 구조랑 필요한 파일들 알려줘
    ```

## 💾 3. 다른 기기로 환경 복제하기 (GitHub Clone & Play)

이 프로젝트는 개인 인증 정보 및 로컬 데이터베이스가 깃허브에 공유되지 않도록 `.gitignore` 안전 처리가 완료되어 있어, 새 컴퓨터로 매우 안전하고 깔끔하게 환경을 이식할 수 있습니다.

### [처음 세팅할 때] - 내 깃허브에 올리기
```bash
git init
git add .
git commit -m "feat: 듀얼 에이전트 및 검토 샌드박스 룰 구축"
git remote add origin [내_깃허브_저장소_주소]
git push -u origin main
```

### [이사 갈 때] - 새 컴퓨터에서 즉시 부활시키기
새 기기나 우분투 환경에서 아래 순서대로 3단계만 진행하면 기존에 조율된 에이전트 두뇌와 환경이 100% 원클릭 복제됩니다.

1.  **새 기기에 `omx` 공식 설치 및 설정**:
    ```bash
    # 공식 가이드대로 설치 완료 후 초기 로그인 진행
    omx setup
    omx login
    ```
2.  **프로젝트 깃 클론**:
    ```bash
    git clone [내_깃허브_저장소_주소] my-project
    cd my-project
    ```
3.  **실행 권한 부여 및 기동**:
    ```bash
    chmod +x scripts/*.sh
    ./scripts/start-ai.sh gemini
    ```

## 🔒 4. 보안 및 로컬 세션 보호 (.gitignore 정책)

본 템플릿의 `.gitignore` 설정은 다음의 핵심 원칙을 강력하게 준수합니다.
*   **공유하는 대상**: `.codex/agents/`(에이전트 규칙), `.codex/prompts/`(지침서), `.codex/skills/`(특수 스킬), `scripts/`(구동 스크립트)
*   **공유 방지 대상**: `.codex/auth.json`(개인 로그인 키), `.codex/*.sqlite`(로컬 대화 DB 히스토리), `.*-pane`(임시 tmux Pane 주소록)
